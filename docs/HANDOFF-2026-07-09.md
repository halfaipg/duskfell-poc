## Run + screenshot recipe

```bash
cargo run -p sundermere-server          # http://127.0.0.1:4107
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless=new --disable-gpu --hide-scrollbars \
  --user-data-dir=/tmp/fresh-profile \
  --window-size=1280,800 --virtual-time-budget=15000 \
  --screenshot=shot.png "http://127.0.0.1:4107/"
```
Fresh profile per shot (stale asset cache otherwise). Server re-verifies
manifests at startup — restart after asset/manifest changes.
`?terrainDebug=material|zone|elevation` colors tiles by property.

Long straight pale ribbons (~15px wide, cream ~rgb(230,220,190)) crossing
the map along tile diagonals, crossing at fixed world points; plus dark
parallel ribbons. Present in EVERY screenshot today including pre-reset.
Ruled out one by one (all still present after disabling):
- material transition overlays (skipped for all patched pairs, then either-side)
- ecology energy/feed link beams + ground effects (hidden)
- relief-edge + side-wall pale strokes (alpha ~0.06 now)
- settlement/road atlas texture (roads are brown earth now; ribbons unchanged)
- NOT the ground patches (predate them)

Remaining suspects, next diagnostic step (was in progress when session
ended): screenshot with `?terrainDebug=zone` and `?terrainDebug=material`
and see what the ribbon tiles are. Note `zone === "ridge"` exists in decal
code — worldgen "ridge" zones drawing something pale? Also check
`drawTerrainHeightShade` positive-shade highlight fill (cream rgba(255,238,178))
— a long straight ridge line of sloped tiles would paint exactly this.
That's my best remaining guess: **ridge-line height-shade highlights**.
